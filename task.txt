Реализовать REST API (серверная часть)

Авторизация - OAuth

API должен реализовывать следующие действия:

1) регистрация пользователя (имя, почта - логин, пароль);

2) авторизацию пользователя с выдачей токена;

3) изменение данных пользователя;

4) добавление авторизованным пользователем набора измерений - матрица географический
 координат на земной поверхности (широта + долгота);

5) получение авторизованным пользователем наборов измерений;

6) вычисление набора с максимальным периметром.

Реализовать REST API клиент работы с серверной частью.

Создание SPA для регистрации, авторизации, редактирования пользователя.
Внесения наборов измерений. Просмотра набора измерений и вычислений наборов с
максимальным периметром.

Использовать:

1) PHP 5.6+;

2) фреймворк Yii/Zend/Symfony/Laravel/CakePHP + любой набор готовых модулей;

3) ООП;

4) Apache/Nginx;

5) PSR2 (http://www.php-fig.org/psr/psr-2/) и PSR4 (http://www.php-fig.org/psr/psr-4/);

6) MySQL/Mongo/Redis/SQLite (выбор должен быть обоснован);

Будет хорошим плюсом, если будет использовано:

- https://getcomposer.org/ для автолоада классов и подключения сторонних библиотек,
используемых для решения задачи (написанных вами в том числе)

- Angular, Knockout, Amber, React для фронта;

- решение в виде ссылки на код проекта на https://github.com/ (в репозитории обязан быть
файл README.md, содержащий инструкцию как установить проект и просмотреть функциональность);

- покрытие код unit-тестами (https://phpunit.de) (юнит тесты добавить в проект с помощью
 composer https://getcomposer.org/)

Доп. задачи:

1) при регистрации пользователя добавлять поля: дата рождения, аватарка - файл или ссылка,
 пол, подписка на рассылку, биография;

2) валидация набора - координаты должны описывать замкнутую фигуру (не содержит точек
внутри периметра).
